Development Environment Prerequisites

Node.js

Some tools require Node.js to be installed. You can test that Node.js is installed by running node -v

Grunt

Grunt is a JavaScript task runner that is used to build the front end templates and assets. Install it by running npm install -g grunt-cli from the command line.

Installing Node.js modules

When you first pull down a project you'll need to get the Node.JS packages which can be done by running npm install from the command line in the root folder of the project. This looks for the package.json file and loads and installs the dependencies specified here.

Bower

Some required components (including core components such as DDBreakpoints) are defined in bower.json and installed via bower. To install these components, first install Bower. Then run bower install from the command line in the root folder of the project.

Running a local development server

In the command line, run grunt dev to start a local Browsersync server at http://localhost:3000. This works together with the watch task, and live reloads the browser when files are changed.

Running individual tasks

The following tasks can be individually run for quick compilation without starting the local development server:

  • grunt html:dev/grunt html:build - Compiles HTML templates
  • grunt css:dev/grunt css:build - Compiles Sass stylesheets
  • grunt js:dev/grunt js:build - Compiles JS files
  • grunt svg - Generates grunticons. See below for more detail

HTML templates

HTML templates are located in source/views/. grunt-include-replace is used to split up the HTML files for maintainability. The structure is as follows:

  • source/views/*.html
    • The file names of these immediate HTML files define the URL which can be viewed in the browser. They donâ€™t contain any contents, but include layouts and corresponding templates contents (see below)
  • source/views/templates
    • The HTML files in here hold the actual contents for each template type
  • source/views/includes
    • This directory contains global elements such as navigation and modal window
  • source/views/layouts/layout.html
    • This file contains the overall HTML structure, such as <html> <head> and <body>

SVG images

Vector graphics are displayed as SVG images so that they are sharp when the page is zooomed. SVGs are not supported on IE8 or Android 2.2 so we use grunticon to fall back to PNG sprites in this case. grunticon takes a folder of SVG files (typically, icons that you've drawn in an application like Adobe Illustrator), and outputs them to CSS in 3 formats: SVG data urls, PNG data urls, and a third fallback CSS file with references to regular PNG images, which are also automatically generated and placed in a folder.

Installing grunticon

  1. Install Node: http://nodejs.org
  2. Install PhantomJS: http://phantomjs.org/download.html
  3. Install Grunt: npm install -g grunt-cli
  4. cd into the root folder of the project (the folder containing config.rb)
  5. Install grunt and grunticon locally (they will be read from package.json): npm install

Getting vectors from Photoshop to SVG

  1. Select the Photoshop layer with the shape
  2. Go to File > Export > Paths to Illustrator
  3. Open the exported AI file in Illustrator
  4. Create artboard for: Artwork bounding box
  5. Select the shape and apply the correct background / line colours
  6. Select the artboard tool (Shift O) and make sure the size of the artboard is appropriate.
  7. Adjust the artboard size and round up (or down) to a full pixel, or otherwise the icon may be cropped in IE9 and IE10! This is also a good opportunity to do some QA on the design. Is there a horizontal line that is 1.01 px tall? Make it a round 1px tall and it will look sharper.
  8. Time to save. Go to File > Save as
    • Location: source/assets/img/svgs-src
    • Format: SVG
    • SVG Profile: SVG 1.1 Tiny
    • Type: Convert to outline before export
    • Images: Embed
    • Don't check "Preserve Illustrator editing" to reduce file size
  9. cd into the root folder of the project and run grunt
  10. To modify the CSS selectors for the image, edit the customselectors property in Gruntfile.js

Saving out icons in multiple colours

Some of the icons need to be exported in multiple colours. We can use grunticon to automate this process, so that we have one SVG in the svgs-src folder which is exported to different coloured SVGs and PNGs in the output svg folder. You define the colours by adding some text to the filename. For example, if you save a file called "ec-collapsed.colors-blue-red.svg", then grunticon will create three files:

  • ec-collapsed.svg (with the original colour)
  • ec-collapsed-blue.svg (shade of blue is defined in Gruntfile.js)
  • ec-collapsed-red.svg (shade of red is defined in Gruntfile.js)
